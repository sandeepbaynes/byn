Features

Everything you need to keep secrets secret

Built for developers who use AI coding agents, run automation scripts, and need credentials to flow transparently.

🔒

Per-terminal sessions

Unlocking in one terminal doesn't unlock for other shells or background scripts. Sessions are bound to the TTY device.

byn unlock
💉

Transparent injection

byn exec uses syscall.Exec to replace itself. Values never appear in shell history, argv, or scrollback.

byn exec -- CMD
🗂️

Scope hierarchy

vault → project → env with inheritance. Override per-environment without duplicating secrets.

--vault --project --env
🪪

Passkey / Touch ID

Enroll from the browser portal. Touch ID for daily unlocks. Your master password stays the recovery root.

localhost:2967
📋

Tamper-evident audit

Every read, write, exec, and reveal logged in an HMAC chain. byn doctor verifies integrity on startup.

byn audit tail
📦

Bulk import / export

Import from .env, YAML, or JSON. Export in any format. Three merge modes: default, skip-existing, replace.

byn import / export
🧱

Privilege separation

Opt in to run the daemon as _byn and pinned byn exec children as _byn-exec — a same-UID non-root process can't ptrace the daemon or read the child's env. Off by default; raises the bar to root (not past it).

byn setup · [security] privsep

Security model

Honest about what it protects

byn stops credential leaks that happen through normal developer workflows. By default it does not protect against a same-UID attacker; opt-in privilege separation raises the bar to root — but not past it. All of this is documented.

  • ✓Shell history / scrollback — values read from stdin, injected via syscall.Exec, never in argv
  • ✓Background script inheritance — sessions are TTY-bound, not process-group inherited
  • ✓Vault at rest — XChaCha20-Poly1305 per row; vault key only ever in daemon RAM (entry names + metadata are plaintext at rest)
  • ✓Same-UID env-sniff (opt-in) — with privilege separation on, a pinned byn exec child runs as _byn-exec; only root / CAP_SYS_PTRACE can read its env
  • ✓Audit tampering — HMAC chain verified on startup; any gap flagged immediately
